## ACs

### AC-1 — Resume on `new-plain` → fresh launch (no --resume)
For `new-plain` tasks, `claude --resume <sessionUuid>` always fails with "No conversation found" because Claude only writes the JSONL transcript AFTER the user's first message. v0.8.7 AC-1 unblocked the Resume CTA on these tasks (idle-on-pty-gone); v0.8.8 AC-1 makes the click actually USEFUL by emitting a fresh `--session-id <uuid>` launch (same task identity, fresh Claude session). Server-side gate at `routes.ts /launch` legacy fallback: `effectiveResume = resume && task.actionId !== "new-plain"`.

### AC-2 — Multi-strategy `resolveClaudeBin()`
Extend lookup with: (1) `SHIPWRIGHT_CLAUDE_BIN` env override (loud reject if path missing), (2) primary `where`/`which` (existing), (3) curated fallback paths per-platform. Windows: `~/.local/bin/{claude.exe,claude.cmd}`, `APPDATA/npm/{claude.cmd,claude.exe}`, `LOCALAPPDATA/Microsoft/WinGet/Links/claude.exe`, `Program Files/Claude Code/claude.exe`. POSIX: `~/.local/bin/claude`, `~/.npm-global/bin/claude`, `/usr/local/bin/claude`, `/opt/homebrew/bin/claude`. Log structured stderr line on every fallback hit for production diagnosability.

### AC-3 — Boot-time PATH self-heal
When AC-2 fallback resolves a binary whose parent dir is NOT on `process.env.PATH`, prepend it. Idempotent (case-insensitive on Windows). Subsequent child-process spawns (node-pty pwsh, preview-session-manager) inherit the augmented PATH. Loud structured log on actual prepend.

### AC-4 — Diagnostic context in `/api/diagnostics`
When `claudeCli.supported === false`, response includes a `diagnostic` block: primary `where`/`which` output, bounded PATH sample (8 entries), curated fallback paths annotated with `(exists)` or `(missing)`. Operators self-diagnose without reading server log. Block omitted on the happy path (no UI noise).

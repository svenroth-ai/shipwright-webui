## Scope for this iterate session (chosen 2026-05-05)

This iterate run covers **Tier 0 minus AC-2**: AC-1, AC-3 + cross-cutting AC-11 (typecheck/lint/tests green) + AC-12 (CHANGELOG drops + UAT checklist). Two release-blocking architectural fixes, ~1.5 days. Ships as **v0.8.1**.

**AC-2 (Black-on-black input) is deferred to a follow-up iterate** — its acceptance criterion mandates "diagnosis-first via WS-frame capture" against a live Claude Code TUI session, which requires running the dev server + an active claude task interactively. That diagnosis can't responsibly happen inside this same code-build session; the resulting theme fix is small (~0.25d) but the diagnosis run is a manual interactive task that belongs in its own iterate. Tracker: `iterate-2026-05-XX-terminal-input-contrast` after v0.8.1 ships.

**Tier 1 (AC-4 Ctrl+V image-paste, AC-5 paste latency, AC-6 awaiting-launch lag) and Tier 2 (AC-7 paste-dir migration + show-ignored, AC-8 replay-only-mode, AC-9 conditional disclosure, AC-10 retention copy) are out of scope for this iterate session.** They will spawn as separate iterate runs after Tier 0 ships, each with their own iterate spec + branch + PR.

Decision rationale: 12 ACs across 3 tiers in one iterate run pushes complexity to LARGE (5–6 day estimate, 13+ files, single PR with merge risk). Slicing keeps each iterate at proper "medium" scope, gives stop-points to ship value sooner, and matches the spec author's `Status: draft (scope-only — re-plan)` invitation.

## Goal

Close the gap between v0.8.0 (which shipped the embedded terminal + auto-launch + disk-backed scrollback) and a UAT-stable terminal experience. UAT 2026-05-05 surfaced **5 new high/critical bugs** that escaped both unit-test and Spec 76 (WebSocket-frame-capture e2e) coverage, plus 3 mini-polish items from the dropped A2 audit.

The bugs cluster around **two architectural seams** that v0.8.0 didn't stress-test:
1. **Replay-on-reattach** — raw ANSI bytes saved to disk get re-executed when re-applied to a fresh xterm. Cursor-control codes (`\e[H`, `\e[K`, `\e[?25l/h`) re-execute as cursor jumps, producing visual chaos and destroying the user's conversation history.
2. **WS writer-promotion under pty load** — the StrictMode reader-cancel race (fixed in v0.8.0 via 1500 ms stability window) is one symptom; the broader issue is that writer-slot transitions are tied to ws.close events that get delayed under high pty-output volume.

Out-of-scope: A2-audit items dropped (`/runtime` endpoint, 8-cell CTA matrix, tab-status-dot, per-task dismissal, AC-17 disabled-mode e2e, global disk-quota + LRU, Smart-Grace 30s/10min split). Custom Actions UI deferred.

## Acceptance Criteria

### Tier 0 — CRITICAL (release-blocking-grade for v0.8.1)

- [ ] **AC-1 (Scrollback replay no longer corrupts TUI applications):** `ScrollbackStore.append()` filters cursor-control sequences from the byte stream before persisting to disk. Color and SGR codes (`\e[31m`, `\e[1m`, etc.) are preserved. Cursor-movement (`\e[H`, `\e[A-D`, `\e[K`, `\e[J`, `\e[<n>;<m>H`, `\e[?25l/h`, `\e[?47/1049h/l`, `\e[s`, `\e[u`) is stripped. The LIVE WebSocket stream remains unfiltered (cursor-codes still drive the live xterm correctly). New unit test in `scrollback-store.test.ts` feeds a PowerShell prompt-repaint sequence (50 redraws of `\e[H...PowerShell 7.6.1\e[K\r\n...PS C:\>...`) and asserts (a) the live-stream subscriber receives all bytes intact, (b) the saved scrollback contains "PowerShell 7.6.1" exactly once and the prompt exactly once, with no `\e[H` patterns. New e2e spec covers re-attach during a TUI-heavy session and asserts xterm buffer length grows linearly with new content (not stacks of overwrites). **Severity: CRITICAL** — currently destroys conversation history on every nav-away+return for any task running Claude Code's TUI or PowerShell. **Estimate: 0.5d**. **Files:** `server/src/terminal/scrollback-store.ts`, `server/src/terminal/scrollback-store.test.ts`, new `client/e2e/flows/77-scrollback-replay-tui-fidelity.spec.ts`.

- [ ] **AC-2 (Black-on-black input rendering eliminated):** Diagnosis-first via WS-frame capture: identify the exact escape sequence Claude Code emits for its TUI input-box that collides with the xterm light theme. Then either (a) extend the xterm theme to remap the offending background ANSI slot to a brand-warm value (similar pattern to the existing white→brand-brown override for invisible-on-beige), OR (b) detect Claude-Code-TUI and switch xterm to a dark theme (`bg=#1a1a1a fg=#f5f0eb`) for the duration of the session. Decision document captures the diagnosis output + chosen approach in `.shipwright/agent_docs/decision_log.md`. New unit test for the theme block asserts no fg/bg ANSI-slot pair has Δ contrast-ratio < 4.5:1 against the chosen background. **Severity: HIGH** — typed input is illegible under Claude Code TUI. **Estimate: 0.5–1d** (diagnosis-first). **Files:** `client/src/components/terminal/EmbeddedTerminal.tsx` theme block (~lines 262–294), `client/src/components/terminal/EmbeddedTerminal.test.tsx`.

- [ ] **AC-3 (False read-only banner under pty-load resolved + pause/resume reference-counting):** Server-side decouples writer-slot eligibility from ws.close-event timing. Two sub-changes: (a) `pty-manager.ts` adds a per-task `pauseResumeRefCount` — `pause()` increments, `resume()` decrements, only the underlying `pty.pause()` fires when refCount transitions 0→1 and `pty.resume()` only when refCount transitions 1→0; closes the multi-tab pause/resume race. (b) `routes.ts` writer-promotion no longer waits for prior conn's ws.close — instead, writer-slot is released when EITHER ws.close fires OR `socket.bufferedAmount` exceeds threshold for >2s (heuristic: stuck connection = release slot). New unit test for refcount semantics + new e2e spec `78-reattach-under-pty-load.spec.ts` that runs a high-volume payload (`for 1..1000 echo`), detaches mid-flow, re-attaches, and asserts no banner + clean rendering. **Severity: HIGH**. **Estimate: 1d** (combined fix — both sub-changes likely share root cause). **Files:** `server/src/terminal/pty-manager.ts`, `server/src/terminal/routes.ts`, `server/src/terminal/pty-manager.test.ts`, new e2e spec.

### Tier 1 — MED (bug fixes, ship in v0.8.1)

- [ ] **AC-4 (Ctrl+V image-paste restored to parity with Alt+V):** Diagnosis-first: WS-frame capture comparing Ctrl+V vs. Alt+V paste paths. Hypothesis: xterm's own keybinding swallows Ctrl+V before the DOM capture-phase listener fires, OR focus-on-canvas vs. focus-on-container affects DOM event delivery. Fix likely involves either disabling xterm's Ctrl+V keybinding (`term.attachCustomKeyEventHandler` returning `false` for Ctrl+V to defer to DOM) or moving the paste handler to `document` level. **Severity: MED**. **Estimate: 0.5d**. **Files:** `client/src/components/terminal/EmbeddedTerminal.tsx` paste handler (~lines 374–443).

- [ ] **AC-5 (Image-paste latency reduced to <1s):** Diagnose the 5s Alt+V paste delay via Browser DevTools Network panel + server-side timing logs. Hypotheses: (a) magic-byte mime-sniff stalling on large images, (b) multipart upload roundtrip slow on Windows file-write, (c) fetch-retry loop. Add structured timing logs around `savePastedImage` covering (i) clipboardItem→Blob conversion, (ii) HTTP roundtrip, (iii) magic-byte sniff, (iv) fs.writeFile, (v) pty.write of the path. Profile, fix the bottleneck. **Severity: MED**. **Estimate: 0.5d**. **Files:** `client/src/components/terminal/EmbeddedTerminal.tsx`, `server/src/terminal/image-paste.ts`, `server/src/terminal/routes.ts`.

- [ ] **AC-6 (Awaiting-launch state lag — diagnosis only, fix conditional):** Add a structured diag log in `core/session-watcher.ts` that records each polled JSONL path + match-result + elapsed-ms-since-launch. Reproduce a slash-command launch (`/shipwright-iterate <prompt>`) and time the awaiting→active transition. **Decision tree:** if <30s, document expected-behavior in `known_issues.md` (claude needs 5–15s for first JSONL write + 2–5s poll cadence = up to 20s total). Pure Claude (`new-plain`) explicitly excluded — JSONL-on-first-message is correct behavior. **Severity: MED (conditional)**. **Estimate: 0.5–1d**. **Files:** `server/src/core/session-watcher.ts`, possibly `server/src/core/launcher.ts`.

## Confidence Calibration

- **Boundaries touched:**
  - Disk persistence format: raw pty bytes → sanitized text (CSI cursor stripped, SGR/CRLF/HT/OSC preserved). WS replay envelope schema unchanged.
  - PtyManager API: NEW `pauseForConn`/`resumeForConn` (per-conn refcount); legacy `pause(taskId)`/`resume(taskId)` retained as anonymous-token wrappers.
  - PtyManager opt-in `watchdogEnabled: true` in production (`server/src/index.ts`); unit tests construct without it for deterministic timing.
- **Empirical probes run:**
  - Sanitizer round-trip: SGR + CRLF + tabs + OSC preserved; cursor-home/erase/alt-screen/save-restore stripped; bare CR + backspace stripped (33 tests).
  - Chunk-boundary carry: CSI split across calls → preserved or stripped correctly per final byte; private CSI carry; OSC carry across BEL or ST terminator; CR-LF split → CRLF preserved; UTF-8 mid-codepoint split → not corrupted.
  - Overflow + recovery: CSI > 32 bytes drains until next final byte or fresh ESC; OSC > 4096 bytes drains until BEL or fresh ESC.
  - Live broadcast unaltered: pty-scrollback-integration "ANSI escape sequences round-trip without corruption" + "survives scrollback append throwing — broadcast loop continues" verify the live path is independent of sanitization.
  - Legacy v0.8.0 file compat: write raw bytes directly to `<taskId>.log` bypassing sanitizer, then `store.read()` returns sanitized output.
  - Refcount semantics: 2 conns each hold a pause stake → only one pty.pause + one pty.resume; idempotent for same conn; legacy anonymous-token API single-fires.
  - Force-evict cleanup: detach() of a paused conn fires pty.resume (no refcount leak).
  - Watchdog timing edges: stuck ≥ 2s evicts; transient (drain mid-window) does not; under-threshold writer never evicted; capability-missing path warns once + disables; watchdog timer cleared on killAll.
  - Stale-close-after-eviction: writer slot identity-check prevents accidental flip.
- **Edge cases NOT probed + why acceptable:**
  - Sanitizer state mid-CSI when rotation fires: sanitizer emits sanitized bytes BEFORE the rotation buffer write; PerTaskState.sanitizer is preserved across rotation (no reset). Manual code-trace confirms; no test added because the integration is implicit.
  - Slow-but-healthy network (high RTT, gradual drainage): may temporarily exceed 512 KiB threshold for ~1s on a 10 Mbps link; the 2s grace covers typical slow-but-draining cases. If UAT shows false-positive evictions, the threshold is constructor-tunable.
  - SGR with truly exotic parameter mixes (e.g. ITU T.416 indexed sub-params with `:`): preserved as-is because we accept any CSI ending in `m` regardless of parameter content. Tested with 256-color and truecolor.
  - Operator-input categories from `boundary-probes.md` (POSIX export, comments, quoted #): N/A — disk persistence format is machine-only, never hand-edited.
- **Confidence-pattern check:** No "are you confident?" → "yes" → bug pattern fired in this run.

## Lessons from v0.8.0 to apply

1. **Stability-window pattern** (1500 ms defer before destructive state-transition) is now repo-canon — apply in any new presence/role state-machine logic. See `client/src/pages/TaskDetailPage.tsx` `handleTerminalReady` for reference.
2. **WS-frame capture in Playwright** (Spec 76) is the strongest e2e tool for this stack — every new spec in this iterate uses it for outcome detection, not console logs or DOM-asserts.
3. **TUI-heavy load conditions** are NOT covered by Spec 76 (which uses cold pty) — every new spec covering AC-1, AC-3 must explicitly produce TUI bytes (PowerShell prompt repaints, Claude Code box-drawing) to exercise the actual failure mode.
4. **StrictMode dev double-mount** + **multi-tab** + **high-volume pty output** are three distinct race conditions that interact — fixes must handle all three together, not in isolation.
5. **Raw-byte persistence is wrong for TUI streams.** Persistence-format must be either (a) printable-text + color codes only (Tier 0 / Option A here), or (b) rendered cell-state via `@xterm/headless + addon-serialize` (deferred). Never the raw byte stream.

## Alternatives considered (rejected)

1. **Disk-pruning of historical scrollback.** Rejected per ADR-076 explicit-rejection ("destructive on persisted user-visible data"). All four ACs stay non-destructive on disk OR add additive markers.
2. **Server-side replay-time filter on the WS routes layer (instead of in scrollback-store.bytes()).** Rejected — `bytes()` is the authoritative reader for ALL replay paths (including `scrollback-meta` size probe, ready envelope `scrollbackBytes` calculation). Filtering only at the WS routes layer would create read-vs-display divergence.
3. **Track pty cycle-rate root cause (Investigation-led iterate).** Deferred — empirical measurement showed dominant source is `tsx watch` SIGTERM during dev (not a production-user concern). The 30-min idle-ceiling alone produces 1-2 banner cycles per day in production, which is acceptable.
4. **Auto-clear scrollback after N markers.** Rejected — user-driven "Clear history" stays the only destructive path per ADR-076.

## Risks + mitigations

| Risk | Mitigation |
|---|---|
| AC-3 collapse swallows legit user content containing the literal "PowerShell N.N.N" | Anchored regex requires the surrounding 10+×CRLF block signature (post-resize fingerprint); 2 unit-fixture tests guard against false positives |
| AC-2 marker breaks xterm rendering on user's color theme | Box-drawing chars (U+2500) + dim ANSI (`\x1b[2m`) are universal; fallback `\x1b[90m` (dark-grey) documented in design notes |
| AC-1 idle ↔ active oscillation when pty respawn races poll | The existing `IDLE_REACTIVATE_THRESHOLD_MS` branch at L879 will absorb it; oscillation possible only within a single poll-cycle window |
| AC-4 footer flickers when N transitions from 1→2 mid-replay | Component reads count from accumulated replay buffer at `replay_end`, not incrementally; flicker not possible by construction |
| Server route factory wiring breaks (createExternalRoutes signature change) | Add `ptyManager` as optional param with `ptyManager?: PtyManager` and fall back to no-op when missing — preserves test isolation |

> **Responsive / tablet (iterate-2026-06-14, FR-01.38).** `client/src/hooks/useIsCompactViewport.ts`
> (matchMedia `(max-width: 1023px)`) is the single compact-band SSoT for the tablet layout (≤1023px).
> `PaneTabBar` (`external/PaneTabBar.tsx`) is the compact task-detail tab switcher (Files·Session·Viewer)
> that `TaskDetailThreePane` renders when compact while keeping ONE persistent `PanelGroup` mounted (the
> embedded terminal is never unmounted across the breakpoint). Desktop ≥1024px is unchanged.
>
> **Responsive / phone (iterate-2026-06-14, FR-01.39).** The same hooks file adds `useIsPhoneViewport`
> (matchMedia `(max-width: 767px)`) and `useCoarsePointer` (`(pointer: coarse)`). Below 768px the sidebar
> becomes an overlay drawer hosted in a Radix `Dialog` by `MainLayout` (phone top-bar hamburger; `SidebarNav`
> gains a `drawer` mode that shares the same nav body with full labels). `TerminalKeyBar`
> (`terminal/TerminalKeyBar.tsx`) is the on-screen key accessory bar `EmbeddedTerminal` renders on coarse
> (touch) pointers — Esc/Tab/Ctrl-C/arrows/Enter written to the pty over the existing
> `socket.send({type:"data"})` writer frame (mode-aware CSI/SS3 arrows). `index.html` adds
> `viewport-fit=cover` + `interactive-widget=resizes-content`; the shell uses `100dvh` + `env(safe-area-inset-*)`.
>
> **Mobile/tablet layout polish (iterate-2026-06-15, FR-01.41).** On the phone band (≤767px) the Task Board
> moves `ProjectFilterDropdown` (new `fluid` prop) into the `MainLayout` top bar via `MobileTopBarSlot`
> (`external/MobileTopBarSlot.tsx` — a `useState`-published portal target so the consumer re-renders, not a
> bare ref), and the status filter collapses to a funnel-icon multi-select menu. `BoardStatusFilter`
> (`external/BoardStatusFilter.tsx`) holds both presentations — `StatusPillRow` (≥768px chips) and
> `StatusFilterMenu` (phone Radix `DropdownMenu`, `preventDefault` keeps it open for multi-select); it was
> extracted out of `TaskBoardPage` to stay under the bloat baseline. On the compact band (≤1023px) the List
> launch CTA is icon-only (`TerminalLaunchButton` compact label `hidden lg:inline`), the Projects `Path`
> column is `hidden lg:table-cell`, the icon-rail count badge overlays the icon (`SidebarNavItem`), and board
> lanes become flexible (basis-0 grow, min-200, longhand-only) so all three fit with no right cut-off; <768px
> keeps the snap carousel and ≥1024px keeps fixed 360px lanes.
>
> **Phone-header polish (iterate-2026-06-15 phone-header-polish, FR-01.41 follow-up).** The top-bar
> `ProjectFilterDropdown` `fluid` variant is content-width (`max-w-[60vw]` + truncate) instead of full-width.
> The All-Projects `+ New` cascade's side-opening submenu overflowed off-screen on phones, so `ProjectCreateMenu`
> branches on `useIsPhoneViewport()` to `ProjectCreatePhoneMenu` (`external/ProjectCreatePhoneMenu.tsx`) — a
> flat downward drill-down (project list → tapping a project REPLACES the popup content with its actions, back
> row returns) reusing the cascade's `ProjectActionsLoader`/`ProjectDot` and the same `onSelect(action,
> projectId)` → NewIssueModal contract. Tablet/desktop keep the nested cascade.

`SmartViewerModal` (iterate-2026-05-31-smartviewer-popout-modal) is the
centered in-app pop-out of a file preview — a Radix Dialog (portal-to-body,
viewport-centered, ESC/backdrop close) reusing the ContinuePipelineModal /
NewIssueModal pattern. The pop-out button in `MarkdownRenderer` opens it via an
`onPopOut` callback instead of `window.open("/preview", "_blank")`; the modal
body renders a nested `<SmartViewer popOut={false}>` so the expanded view shows
no further pop-out button. The `/preview` route is retained for a standalone
browser tab/window.

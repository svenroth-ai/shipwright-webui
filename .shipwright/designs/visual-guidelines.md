## Typography

- **Primary font:** Inter (loaded as `'Inter', sans-serif`; weights 400 / 500 / 600 used)
- **Monospace:** `ui-monospace, SFMono-Regular, Menlo, Consolas, monospace` â€” fenced code, inline code, transcript ANSI output

## Spacing & Layout

- **Base unit:** 4px (Tailwind default; webui doesn't override)
- **Horizontal page padding:** 24px L / R (`.page-container`, `.board-container`)
- **Page max-width:** 1280px for content-heavy pages (Inbox, Projects, Settings, Diagnostics); 1600px for the kanban Board
- **Card padding:** 22px Ã— 24px in "comfortable" surfaces (Inbox / Transcript / TaskCard); 16-20px in dense list rows

## Component Patterns

- **Buttons:**
  - **Primary** â€” brown `#6b5e56` solid, white text, 8px radius, `--shadow-sm`. Hover â†’ `#5a4f48`. Used for `New task`, `Create Project`.
  - **Secondary** â€” surface bg with `--color-border` border, foreground text, 8px radius. Used for `Board` / `List` toggle, action menu items.
  - **Resume** â€” soft terracotta `#C08862` solid, white text, 8px radius. Used only on Resume action (distinct from primary + warning, ADR rationale: brown=Terminal, orange=Warning, terracotta=Resume).
  - **Icon-only** â€” square 32-36px, transparent bg, hover â†’ muted-bg tint. Used for "Plain Claude", "Task actions", "More create options" split-button caret.
- **Cards:**
  - 12px radius, white surface, default `--shadow-card`, no visible border (shadow does the lift).
  - TaskCard has hover state: `--shadow-card-hover` + 1-2px translate.
  - Master TaskCard renders child phase-tasks inline below the parent body, separated by a `--color-border` divider.
- **Navigation:**
  - Fixed dark-beige sidebar (`#5c5652`) on the left, white text + icons.
  - Active route â†’ 1px white left-border + slightly brighter row background.
  - Sidebar entries: Task Board, Projects, Inbox, Diagnostics; Settings as a separate bottom-pinned link.
  - "+ New â–¾" split-button at the top of the sidebar (Pipeline / Iterate / Task / Continue Pipeline).
- **Forms:**
  - Labels above inputs, 13px / weight 500, foreground color.
  - Inputs: white bg, 1px `--color-border` border, 8px radius, 10-12px vertical padding, focus â†’ 2px primary-tint ring + border darkens to `--color-accent`.
  - Validation: error text in `--color-error`, error inputs gain a `--color-error` border.
- **Tables:**
  - Header row on `--color-muted-bg`, weight 600.
  - Cell padding 6px Ã— 13px (markdown tables); 10-14px in UI tables (Diagnostics SESSIONS, Components inventory).
  - Alternating rows tinted `rgba(175, 184, 193, 0.08)` for legibility.
  - 1px `--color-border` cell + outer borders.
- **Status chips:**
  - Small pill, full radius, 11-12px / weight 500.
  - Color encodes phase or state: phase chips use the phase palette; state chips (`draft`, `awaiting`, `active`, `idle`, `done`, `launch-failed`, `jsonl-missing`) use neutral muted-bg + foreground or success/error pairs as appropriate.
- **Bubble surfaces (Inbox + Transcript + comfortable cards):**
  - 22px Ã— 24px padding, body 15px / weight 600 / line-height 1.45.
  - Markdown-rendered content scoped under `.markdown-body` (GitHub-style with warm-beige re-skin).
- **Scrollbars (themed):**
  - Width 6px, thumb `--color-border`, hover thumb `--color-accent`, transparent track.
  - Applied globally via `html, body` + `*::-webkit-scrollbar`; opt-in utility `.scroll-themed` for nested overrides.

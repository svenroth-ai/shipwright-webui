Triage Inbox no longer loses findings to a damaged log line. When two triage records ended up on the same physical line (a writer left no trailing newline, so the next append landed on it), the reader skipped the whole line and BOTH findings silently disappeared from the Inbox. Records on such a line are now recovered in order, and the writer checks for an unterminated line before appending so the situation stops arising. Ports the same fix already shipped on the Python side, restoring cross-language parity.

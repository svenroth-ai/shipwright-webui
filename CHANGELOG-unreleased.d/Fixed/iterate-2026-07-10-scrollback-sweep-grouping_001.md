The daily/boot cleanup that deletes old terminal scrollback files no longer throws away a still-active task's saved history (campaign webui-deep-audit D16, F25). Each terminal task keeps its scrollback in two files: a live file and one rotated archive. The cleanup is meant to skip a task entirely if either file has been touched recently. Previously it decided file-by-file in the order the operating system happened to list them, and for the common alphabetical order it visited the fresh live file first, harmlessly, and then still deleted the older archive of that same task — so a task you had used recently could silently lose the earlier part of its terminal history. The cleanup now groups both files of a task together and skips the whole task whenever either file is recent, regardless of listing order, and still removes a fully-expired task in one step. Nothing visible changes for current tasks; the fix only prevents premature deletion of a live task's archived scrollback.

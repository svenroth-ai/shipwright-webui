Terminal copy no longer silently fails after you select text. In Claude's mouse-tracking TUI every mouse move makes the app redraw, and an xterm redraw wiped the live selection a moment before you pressed Ctrl+C — so Ctrl+C found nothing to copy and became a SIGINT. The terminal now captures the selection the instant you finish selecting and holds it, so Ctrl+C / Ctrl+Insert copy it reliably (via the `execCommand` fallback, so it works over plain http/Tailscale too). A one-click **"Auswahl kopieren"** button also appears by the selection for a mouse-only copy — no chord, no need to release Shift. Copy-on-selection stays opt-in / off (no clipboard clobber), and a typed key or a fresh selection clears the captured text so Ctrl+C still interrupts when nothing is selected.

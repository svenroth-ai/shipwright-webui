Production deploy no longer strands the machine with no server. Run from the Command Center's embedded terminal, `start-server-production.{ps1,sh}` was a descendant of the Hono server it killed, so the kill took the script down before it could start the new build (fresh build on disk, no server, no error message). Kill + start + readiness + `~/.claude.json` heal now run in the detached `scripts/deploy-swap.mjs`, which survives the cascade and writes its verdict to `~/.shipwright-webui/deploy-status.json`; a kill that fails now aborts the deploy instead of leaving nothing running.

TaskDetail header (`LaunchCTA`, `ResumeCTA`, `HeaderMenu`): the transient "Launching…" / "Resuming…" / menu-notice label reset timers are now cleared on unmount. Previously each `flashCopied` / `flashMenuNotice` scheduled a `setTimeout` with no cleanup, so a timer firing after the component unmounted ran a `setState` on a dead component — under Vitest this surfaced as a flaky `ReferenceError: window is not defined` unhandled error after jsdom teardown (observed in CI on PR #79). Added a `useEffect` unmount cleanup to all three components; covered by RED→GREEN regression tests in each component's test file.

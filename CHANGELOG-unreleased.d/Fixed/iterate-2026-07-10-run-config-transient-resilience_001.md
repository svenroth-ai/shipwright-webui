The pipeline progress card no longer disappears from the Command Center board when a single background read of a project's run-config file is briefly interrupted mid-run (campaign webui-deep-audit D10, F15). On Windows the orchestrator rewrites shipwright_run_config.json on every phase transition, and one poll could catch that rename window: the server returned an "invalid" result and the board latched its 5-second polling OFF, so the Pipelines lane vanished and only came back after switching windows or navigating away. The read now retries the same brief filesystem hiccups on the existence check (not just the file read) and keeps a valid copy for 30 seconds so a torn read never surfaces as invalid, and the board keeps polling on a light 10-second backoff so any transient flap heals on its own. Genuinely-missing or malformed configs behave exactly as before.

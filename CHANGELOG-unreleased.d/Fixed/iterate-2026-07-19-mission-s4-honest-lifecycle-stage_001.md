The Mission tab's **"Where it stands"** now follows what a session is really doing instead of jumping ahead. Before, the first time a session wrote *any* file that was not a specification, the tracker jumped to **Build** — so writing a scratch note or a throwaway probe script while still reading the code and working out the plan made it look like building had started. Checking 114 real sessions, 15% did exactly that. Now a throwaway write — scratch space, a plan or to-do file, internal bookkeeping, a memory note — counts as part of the groundwork, so **Analyze holds while the session is still scouting and calibrating**; editing real project code still moves it on, because pretending otherwise would be the opposite mistake. The tracker also no longer slides **backwards**: a session that had already opened its pull request used to fall all the way back to "Analyze" as soon as it saved a note afterwards, and now correctly stays at the end. **Spec** now means the specification was actually written, and a command is only counted when it was actually run: looking something up — checking an existing pull request, opening the changelog, reading a test config — no longer claims that stage, and neither does a message that merely mentions a command. Editing a file whose *name* happens to contain "changelog" no longer counts as finishing up. The one-line "what it's doing" summary now reports the **latest** action rather than the furthest-along one, so it can no longer say "Running tests" while the panel beside it says the session is reading code. The tracker fits the kind of work being looked at: a pipeline step shows the phase its run configuration actually records (and an honest "—" if that cannot be read, rather than falling back to a guess); a campaign shows the unit being worked on now, never a finished earlier one; and an ordinary Claude session no longer shows the six-stage tracker at all, since it is not running that process — it gets a plain line saying what it is doing. A session whose transcript is missing or unreadable shows "—" rather than defaulting to "Analyze" (FR-01.66).

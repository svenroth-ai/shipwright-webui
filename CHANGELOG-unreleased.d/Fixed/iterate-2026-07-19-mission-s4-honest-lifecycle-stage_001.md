The Mission tab's **"Where it stands"** now follows what a session is really doing instead of jumping ahead. Before, the first time a session wrote *any* file that was not a specification, the tracker jumped to **Build** — so writing a scratch note or a throwaway probe script while still reading the code and working out the plan made it look like building had started. Checking 114 real sessions, 15% did exactly that. Now a throwaway write — scratch space, a plan or to-do file, internal bookkeeping, a memory note — counts as part of the groundwork, so **Analyze holds while the session is still scouting and calibrating**; editing real project code still moves it on, because pretending otherwise would be the opposite mistake. **Spec** now means the specification was actually written: merely running a command with "plan" or "spec" in its name no longer counts. Only real recorded actions count — a message that just mentions a command cannot move the tracker. The tracker also fits the kind of work being looked at: a pipeline step shows the phase its run configuration actually records (and an honest "—" if that cannot be read, rather than falling back to a guess); a campaign shows the unit being worked on now, never a finished earlier one; and an ordinary Claude session no longer shows the six-stage tracker at all, since it is not running that process — it gets a plain line saying what it is doing. A session whose transcript is missing or unreadable shows "—" rather than defaulting to "Analyze" (FR-01.66).

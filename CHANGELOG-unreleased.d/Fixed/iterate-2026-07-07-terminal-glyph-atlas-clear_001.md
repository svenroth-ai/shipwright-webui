Embedded terminal: fixed the recurring glyph corruption ("wrong / transposed letters") that appeared in long sessions and only a manual resize healed. The WebGL renderer swaps its glyph texture-atlas whole on a terminal-option change (notably the live light/dark theme re-resolve, FR-01.44) via `_handleOptionsChanged` **without clearing the render model**, so cells kept coordinates into the old atlas layout and `term.refresh` — which skips cells that "look unchanged" — could never undo it. The atlas-heal now calls `term.clearTextureAtlas()` (the public equivalent of the resize heal; it clears the render model) on `onChangeTextureAtlas` + `onRemoveTextureAtlasCanvas`, deferred to one coalesced microtask (safe against the mid-merge re-entrancy) and deliberately not on `onAddTextureAtlasCanvas` (append-only — clearing there would feedback-loop). Real-browser proof: flow spec 94 (renderer=webgl, atlasRepaints=5, bounded).

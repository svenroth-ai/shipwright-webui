A dev server you start inside the embedded terminal (for example npm run dev for another app) no longer accidentally inherits the Command Center's own network port settings and clashes with it (campaign webui-deep-audit D12, F17). On a deployed server the Command Center is launched with PORT set to 3847, and every embedded terminal used to inherit that value plus its sibling VITE_PORT / HONO_HOST settings - so a port-honouring dev server started in the terminal tried to bind the same 3847 and collided. The embedded terminal now starts with a clean network slate, so your dev server picks its own default port. This is a terminal-environment hardening fix only; nothing else in the Command Center changes.

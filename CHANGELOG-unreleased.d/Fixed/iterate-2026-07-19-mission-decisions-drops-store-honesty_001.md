The Mission tab's **Decisions** panel now shows what a run actually decided, instead of staying empty until the next release. A finished piece of work records its decision immediately, but the numbered entry in the project's decision log is only written later, when a release is published — so in between, Decisions was reading a place that is empty *by design* and showing nothing. On this project that was not an edge case: **all 18 unreleased runs had a recorded decision that the panel could not see.** Decisions now reads both places. A decision that has been made but not yet published in a release is shown as exactly that — real and recorded, clearly marked **"Decided — not yet published in a release"**, and deliberately without a decision number, because it has not been given one yet and inventing one would let someone quote a number that does not exist. Once a release does publish it, the numbered version is what you see. A decision now also appears **while the work is still running**, rather than only after it finishes. If a release published only *some* of a run's decisions — which can happen, because decisions are published one at a time — the ones already published and the ones still waiting are now **both** shown, each labelled correctly. Previously the moment any decision had been published, the rest stopped being looked for at all, and a decision still waiting simply disappeared with nothing to indicate it. Throughout, "we could not read the records" and "nothing was decided" are kept apart: an unreadable record says so plainly, a genuinely empty one stays quiet, and a single damaged record no longer hides the good ones alongside it — those still show, with a note saying how many could not be read. Separately, **campaign progress stops presenting a guess as fact**: when a campaign's live status file is missing or damaged, the panel falls back to the campaign's plan document as before, but now says so in plain words instead of stating a stale row as the current situation — so "this unit is running now" can no longer be read off a document written days earlier without any indication. And when the records are damaged badly enough that nothing can be counted, the panel says the progress could not be established, rather than reporting the campaign as empty. Where a unit is not mentioned in *any* of the campaign's records, the panel now says its state is an assumption rather than something that was reported — previously it credited that guess to the campaign's plan document, which had not mentioned the unit either (FR-01.66).

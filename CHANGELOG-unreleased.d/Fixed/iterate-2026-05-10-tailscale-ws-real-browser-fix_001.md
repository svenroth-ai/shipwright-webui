Embedded terminal + Resume click now actually work over Tailscale MagicDNS (`http://<host>.tail<id>.ts.net:5173`). Previous v0.9.0 commit `f852a36` wired the SHIPWRIGHT_NETWORK_PROFILE flag into the HTTP CORS middleware, but the WebSocket upgrade gate in `terminal/routes.ts` still called `resolveTrustedOrigins(process.env)` without the tailscale-exec parameter — so it silently fell back to loopback-only and rejected every non-loopback Origin with 500. Symptom: terminal pane stayed empty, Resume button did nothing. Fix: index.ts now passes the boot-time-resolved policy as `deps.allowedOrigins` into `createTerminalRoutes`, making HTTP and WS gates share ONE policy instance. Empirically verified via real Chromium browser test against the Tailscale MagicDNS URL (ADR-083).

The Command Center no longer prints a false "preview misconfigured" warning at startup for backend-only projects (campaign webui-deep-audit D23, F32): a stack profile that intentionally declares an empty frontend (`stack.frontend: {}`) with no dev server — like the bundled python-plugin-monorepo profile used by the shipwright monorepo itself — was being treated as "has a frontend" because an empty object counts as truthy, so the boot-time preview-capability coherence check logged a spurious "profile declares stack.frontend but no dev_server.command" warning on every single server start. An empty frontend declaration is now correctly read as backend-only, so the warning stays quiet for these projects while still firing for a genuinely misconfigured profile (a real, populated frontend that is missing its dev-server command). The Preview button's own visibility gate was already correct and is unchanged — only the noisy, misleading startup log is fixed.

Re-opening a running pipeline from the board no longer fails with Claude's "Session ID already in use" (campaign webui-deep-audit D18, F14 + F28). When you click Launch/Resume on a single-session pipeline card just after it started — within the few seconds before the Command Center has noticed the new conversation on disk — the app used to build a fresh-start command that Claude rejected as a duplicate. It now checks the actual conversation file on disk and correctly builds a Resume command instead. The same fix covers re-clicking Continue on a multi-session phase whose session had already started but died before it was claimed. The card's button also now flips to "Resume" the instant the conversation appears on disk, so its label matches what will actually run. Fresh, never-run tasks are unaffected — they still start cleanly.

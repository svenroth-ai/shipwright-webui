The Windows installer now sets up the "start on login" launcher correctly even when your Command Center folder lives under a path that contains accented or non-English characters - for example C:\Users\Müller\Documents\shipwright-webui (campaign webui-deep-audit D13, F18). Previously the installer saved that background launcher file in a plain-ASCII format that replaced every such character with a "?", so the login autostart silently pointed at the wrong folder and never started - while the installer still reported "Installation complete!". The launcher is now saved in a Unicode format that Windows reads natively, and the installer re-reads it and stops with a clear error (instead of a false success) if the folder path could not be preserved. This is a Windows-setup fix only; nothing in the running Command Center changes.

The developer "restart the dev server" command (`npm run dev:fresh`) no longer risks killing your open browser tab or an unrelated program that happens to sit on a similar-looking port (campaign webui-deep-audit D11, F16). The script used to search the raw port listing for anything containing the port number, which also matched a browser tab already connected to the app and any process on a look-alike port such as 51730 versus 5173 - so it could force-close them too. It now targets only the actual dev-server processes that are genuinely listening on the two configured ports (and correctly finds IPv6 ones, which the old command skipped). This is a developer-tooling fix only; nothing in the Command Center itself changes.

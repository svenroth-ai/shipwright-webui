Embedded terminal: selecting text with the mouse no longer silently overwrites the OS clipboard. Copy-on-selection was always on and clobbered the item you were about to paste — with Windows Clipboard History (Win+V) the previous entry would resurface on a right-click paste and look like a "double paste". Copy-on-selection is now opt-in (default off, VS Code parity); explicit Ctrl+C / Ctrl+Insert copy is unchanged. Added real-browser regression guards: a selection no longer writes the clipboard by default, and a single paste reaches the pty exactly once (both the Ctrl+V key path and the native paste-event / right-click path).

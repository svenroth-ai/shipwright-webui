Terminal copy works again. Claude Code copies your mouse selection itself via the terminal's **OSC 52** clipboard escape, which the embedded terminal now honors — decoding it and writing it to your OS clipboard, with an `execCommand` fallback so it works over plain **http/Tailscale** too (and without stealing the terminal's keyboard focus). Clipboard *read* requests are refused, so a program can never read your clipboard back through the terminal. The WebUI's own copy shortcuts were removed as redundant: **Ctrl+C / Ctrl+Insert now pass straight through as interrupt** (previously they could swallow a real interrupt when text was selected), and copy-on-selection (plus its Settings toggle), the redraw-proof "Copy" pill, and the "Maus-Modus aktiv" hint are gone. Paste (Ctrl+V / Shift+Insert / right-click) is unchanged.

Embedded terminal: touch-scroll now works inside Claude Code's TUI (and every other DECSET-1049 alt-screen TUI: vim, less, htop). One-finger pan is routed by xterm buffer type — alt-buffer pan emits Cursor-Up/Down keystrokes to the pty via the same WS data-channel that user keypresses take, so the TUI scrolls itself; normal-buffer pan still uses `term.scrollLines()`. Closes the regression empirically reproduced in ADR-131 / PR #110.

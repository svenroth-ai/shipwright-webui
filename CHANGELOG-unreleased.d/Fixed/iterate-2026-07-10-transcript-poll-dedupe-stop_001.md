The Transcript view for a finished task no longer keeps re-reading the session file forever, and each open task page now makes a single transcript request per second instead of two (campaign webui-deep-audit D15, F21 + F22). Previously, opening a task that was already done (or had failed to launch) and leaving the tab open kept the app polling that task's transcript once a second indefinitely, even though it was meant to stop once the task reached a final state. Separately, every open task page ran that once-a-second read twice — the page body and the header each polled on their own — doubling the disk reads and network payloads per open tab. Now the poller stops as soon as the task is done or launch-failed, and the model name shown in the header comes from the single shared poller rather than a second one. Nothing visible changes: the transcript, the model label, and live updates all behave the same, just with less wasted work.

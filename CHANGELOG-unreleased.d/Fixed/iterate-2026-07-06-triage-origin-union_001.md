Triage board: dismissed items no longer reappear ("ghosts") when the local checkout is behind origin. The board read now unions the delivered-origin snapshot (`git show origin/<upstream>:.shipwright/triage.jsonl` — read-only, cached by ref SHA, kept fresh by a single-flight, non-interactive, timeout-bounded background fetch) with the local tracked ∪ outbox log, so a dismiss already delivered to origin is reflected before a manual `git pull`. The resolution algorithm (`readAllItems`) is unchanged; a new additive composer handles the third source. Degrades to the prior local-only read on any git failure (never worse than before) and is gated by `SHIPWRIGHT_WEBUI_TRIAGE_ORIGIN_UNION` (default on). A staleness banner surfaces on the Triage tab when the checkout is behind origin.

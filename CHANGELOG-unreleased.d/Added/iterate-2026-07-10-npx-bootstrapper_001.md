`npx @svenroth-ai/shipwright@latest` — one command to install **and** update the whole system (the `/shipwright-*` SDLC plugins **and** the Command Center), first run and every run after. A new `bootstrapper/` npm workspace ships the built server + client, so re-running the command is the update (no clone, no `make`, no `git pull`): it verifies prerequisites (claude, uv, a working Python — rejecting the Microsoft-Store `python3` alias stub — Node ≥ 20.12, git), installs/updates every plugin **read from the marketplace manifest** (never a hardcoded list — the README said "13" while the manifest lists 14) and syncs the plugin cache so each plugin's hooks actually resolve (`claude plugin install` alone never delivers the `shared/` tree, leaving every hook dead at session start), then boots the Command Center on `:3847` — or attaches to a running one, swapping an older one in place via a detached swapper so it is safe to run from inside the Command Center's own terminal, and leaving any foreign process on the port untouched. `/api/diagnostics` now additively reports the WebUI's own `app.name` + `app.version` (the attach-vs-swap signal). Publishing stays a manual maintainer step (FR-01.49).

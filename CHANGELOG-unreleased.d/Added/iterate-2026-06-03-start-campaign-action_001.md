Triage "Start Campaign" action: a campaign-umbrella triage item now has a one-click button that activates its campaign (`draft → active`) and jumps to the Task Board, where the now-active campaign appears. Previously the only button was "Fix now", which launches a single iterate — the wrong action for a campaign (it is demoted to secondary on these items). New `POST /api/campaigns/:projectId/:slug/start` performs the single, lock-protected campaign-state write the WebUI makes (status.json top-level `status`, or the `campaign.md` frontmatter when no status.json exists); all other campaign writes still belong to the framework. Already-complete campaigns can't be reverted; already-active ones show "Go to board".

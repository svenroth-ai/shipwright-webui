Mission tab now answers for **pipeline steps and campaigns** in their own right, instead of falling back to the finished-run rail. A pipeline step shows what it is and where it stands — which phase, which part of the work, whether it is running, finished or failed, when it started, how many attempts it took and what it recorded producing — plus a link to the project specification that holds its requirements. It is matched on the step's own id, so a run with several parts in the same phase can never show one part's work under another. A campaign shows its brief, its runbook and its progress separately from the **one unit currently being worked on**, so a finished unit's commit or test result can never be mistaken for the campaign's own; the current unit is chosen by a stated rule (whatever is running, otherwise the first unfinished one — including one that failed or was escalated for a decision — otherwise the last), and the Mission tab tells you which rule applied. A unit that has not reported its tests yet reads "not recorded" rather than "0 of 0 passed". A campaign whose records cannot be read says so instead of appearing empty, and progress updates as a campaign advances without needing a reload. The Mission tab is hidden entirely only for a project genuinely set up with its own custom actions and no Shipwright pipeline; a project whose actions file is malformed, truncated, empty, unreadable, the wrong shape, or which also runs the pipeline keeps its Mission tab — hiding a working feature with no error and no explanation is the worse mistake. If the tab disappears while you were looking at it, you land on Files & Terminal rather than an empty panel. Left-panel typography now separates headings from their values properly (FR-01.66).

The design-approval step now lives inside the task's Mission view instead of a separate screen. When a run pauses for your sign-off, the Mission view shows the same three-card layout as everywhere else: the Record on the left (with "Design" marked as the step you're on), a gallery of the actual proposed screens in the middle — each a real, clickable preview, not a grey box — and one clear "Approve — start building" button next to "Request changes". Approve confirms in two clicks so you can't fire it by accident, and it runs the exact same Resume action as always (the Command Center never drives Claude itself). While you're deciding, the competing "Resume" button in the header steps aside so there's only one primary action, and the Tests/Serves badges stay honestly empty (a design gate happens before any tests exist). If no previews were produced, you get an honest "nothing to show yet" message rather than a fake gallery. "Request changes" writes your feedback exactly as before. The gate clears itself the moment the run moves on — no refresh needed.

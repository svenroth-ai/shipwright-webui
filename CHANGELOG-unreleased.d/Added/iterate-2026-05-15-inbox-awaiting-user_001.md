Inbox now surfaces plain-text "how should I proceed?" questions from the embedded terminal, not only `AskUserQuestion` tool calls. When a session's latest turn is a turn-ended assistant message asking the user a question — text ending with `?`, or holding a numbered/lettered option list — it appears on `/inbox` as a `text_question` card that auto-clears the moment the user replies in the terminal (no dismiss action). `GET /api/external/inbox` items gain a `kind: "ask_tool" | "text_question"` discriminant; `text_question` items carry the detected `questionText`. New `detectAwaitingUserQuestion()` / `deriveSessionInbox()` in `server/src/core/inbox-derive.ts`. (iterate-2026-05-15-inbox-awaiting-user)

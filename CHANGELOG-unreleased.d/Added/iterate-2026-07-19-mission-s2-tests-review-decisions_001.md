Mission tab now completes the artifact rail with **Tests · Review · Decisions**, alongside the Spec · Requirement · Commit that landed with the resolver. **Tests** reads the run's own commit as a baseline diff and classifies each changed test file as added, changed or removed — a removed test is caught precisely because the diff, unlike the current traceability manifest, still remembers it — then enriches each row with its layer and the requirements it covers, showing "mapped from FR-01.44" wherever a folded requirement id was resolved to its surviving parent. **Review** reports the four passes (plan, code, doubt, external code) from the per-iterate review records, and anything it cannot actually read is shown as unread rather than as clean — both a pass with no record at all, and a pass that ran but whose findings count was never written down. **Decisions** shows only the ADRs tagged with this run's own id, so a second iterate running at the same time can never leak its decisions into yours. Every source is size-bounded, and a source that cannot be read says "currently unavailable" instead of reporting "no tests", "no findings" or "no decisions" (FR-01.66).

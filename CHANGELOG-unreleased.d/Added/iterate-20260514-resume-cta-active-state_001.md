Surface Resume CTA on TaskDetail header + kanban TaskCard for every `(state=idle | state=active)` task, regardless of `liveSession`. The earlier ADR-095/ADR-096 `liveSession` gating was falsified empirically: the signal only checked "a pty entry exists in PtyManager", not "Claude is in the pty foreground". The most common stuck-state was the misfire — Claude TUI exited but the parent shell (pwsh) survived, leaving `liveSession=true` and Resume hidden, so the user had no UI path back even though `claude --resume <uuid>` into the live shell would have done the right thing. Single "Resume" label across all recovery scenarios (no Recover-vs-Resume differentiation).
